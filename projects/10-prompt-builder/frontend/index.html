Example: A Well-Structured Prompt

This is what a completed prompt looks like when all sections are filled in.

ROLE Act as a senior AWS Solutions Architect with expertise in serverless architecture, cost optimization, and enterprise migration projects. TASK Design a migration plan for moving a legacy three-tier web application from on-premises infrastructure to a fully serverless architecture on AWS. CONTEXT The application is a .NET-based inventory management system used by 500 warehouse employees across 3 locations. It currently runs on two Windows servers with an SQL Server database. Peak usage is Monday-Friday 6 AM to 6 PM EST. The company has a $50,000 annual AWS budget and needs zero downtime during migration. CONSTRAINTS - Use only AWS-native services (no third-party tools) - Migration must complete within 90 days - Maintain backward compatibility with existing barcode scanners - Do not exceed $4,000/month in AWS costs - All data must remain in the us-east-1 region OUTPUT FORMAT Return a phased migration plan with: phase number, phase name, duration in weeks, AWS services used, estimated monthly cost, risk level (low/medium/high), and rollback procedure for each phase. End with a total cost summary and timeline. VERIFICATION Before responding, confirm you understand the current architecture, budget constraints, and zero-downtime requirement. EXAMPLE Phase 1: Database Migration (Weeks 1-3) - Services: DMS, Aurora Serverless v2 - Monthly cost: $800 - Risk: Medium - Rollback: Revert DMS replication, point app back to on-prem SQL Server
